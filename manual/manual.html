Section 1

Introduction

What Ca²⁺tchAll does and how the interface is organized

1.1 What is Ca²⁺tchAll?

Ca²⁺tchAll is a browser-based analysis and visualization tool for multi-replicate calcium imaging experiments. It takes .xlsx files exported from the Ca²⁺tch-One and instantly produces publication-quality superplots — charts that overlay individual replicate distributions alongside summary statistics.

The workflow is straightforward: upload your files, type a condition name next to each one, and the plots appear automatically. Everything runs locally on your computer — no data is ever sent to an external server.

ℹ What is a superplot?
A superplot shows both the underlying data distribution and the replicate-level summary in the same chart. Individual ROI values are shown as small dots; per-experiment means are shown as larger dots; the overall condition mean ± error bar is drawn on top. This is the format recommended by Lord et al. (2020) for transparent reporting of biological replicates.

1.3 Getting Started in 3 Steps

  1. 1
    Start the server — Run start.sh (Linux/Mac) or start.bat (Windows) from the Ca²⁺tchAll folder. Your browser will open automatically at http://localhost:5000.
  2. 2
    Upload your files — Click Add Files in the top header, or drag-and-drop .xlsx/.xls files onto the Experiments panel. Wait for the upload progress indicators to show "completed".
  3. 3
    Assign conditions — Type a condition name (e.g. "Control", "Treated") next to each file. Files sharing the same condition name are grouped as biological replicates. Plots appear automatically as soon as at least one condition is assigned.
💡 First-time tip
On your first visit, an interactive 5-step onboarding tour launches automatically. You can skip it at any time and replay it later by clicking the ? button in the bottom-left corner.

Loading Files

Uploading experiments, assigning conditions, and managing session memory

2.1 Supported File Format

Ca²⁺tchAll reads .xlsx and .xls workbooks exported by the Ca²⁺tch-One. The workbook must contain at least one of the following sheets:

Sheet nameRequired columnsContent
Metricsroi_id + metric columnsOne row per ROI — scalar values for each measured metric
Raw_TracesTime_s + ROI_* columnsRaw fluorescence intensity time series per ROI
DeltaFTime_s + ROI_* columnsNormalized ΔF/F₀ time series per ROI
Metadata (optional)Key–value pairsExperiment metadata (signal mode, etc.)
Settings (optional)Key–value pairsAcquisition settings (analysis mode, etc.)
⚠ Trace sheet size limit
Trace sheets with more than 500,000 cells (frames × ROIs) are automatically skipped with a warning to prevent browser slowdown. The Metrics sheet is always read in full regardless of size.

2.2 Upload Workflow

There are two ways to add files:

  • Click "Add Files" in the top header bar — opens your operating system file picker. You can select multiple files at once.
  • Drag and drop — drag one or more files from your file manager directly onto the Experiments panel on the left. The panel highlights with a blue border when a drag is detected.

Each file goes through the following stages shown in the upload job list:

Key points about condition assignment:

  • Same name = same condition: Files sharing an identical condition name are grouped as biological replicates of that condition (e.g. three files labeled "Control" form one condition with three replicates).
  • Autocomplete: As you type, existing condition names appear as suggestions in a dropdown — click to apply.
  • Auto-restore: If you have uploaded the same filenames before and assigned conditions, those assignments are restored automatically on re-upload. A Restored: [name] badge is shown.
  • Plots update instantly: Each time you change a condition name, all plots refresh within a fraction of a second.
  • Remove a file: Click the × button on any file row to remove it from the session immediately.

2.4 Session Memory Bar

Session memory 450 MB / 1000 MB
45% used

The session memory bar in the top header tracks the total in-memory size of all loaded files. It updates after every upload or deletion.

  • Normal (0–70%): All good — continue loading files.
  • Warning (70–90%): The bar turns orange. Consider removing files you no longer need before adding more.
  • Danger (90%+): The bar turns red. New uploads will be blocked if they would exceed the memory cap.

The memory cap is automatically calculated as 25% of your system's total RAM (minimum 512 MB), ensuring the app never destabilizes your machine.

Section 3

Metric Tabs

Understanding each calcium imaging metric and the trace visualization tabs

The tab bar at the top of the plot area shows one tab per available metric. Tabs are shown only for metrics present in the loaded files — if no file contains TG data, for example, those tabs remain hidden. The active tab is highlighted and its plot is shown in the main area.

3.1 Standard Metrics

These metrics describe individual calcium events detected in the recording:

TabFull nameUnitWhat it measures
Peak Peak amplitude ΔF/F₀ The maximum height of a calcium transient above baseline. A larger peak means a stronger calcium response.
AUC Area Under the Curve ΔF/F₀ · s The total calcium flux over the duration of an event (integral of the signal). Reflects both amplitude and duration combined.
FWHM Event Full Width at Half Maximum s How long an event lasts measured at 50% of its peak amplitude. A longer FWHM means the calcium signal persists longer.
Frequency Event frequency Hz How many calcium events occur per second in a cell. Higher frequency indicates more active calcium signaling.
TTP Time to peak s How long it takes from event onset to reach the maximum amplitude. Reflects how fast the response rises.
Decay Decay t½ s The half-life of the falling phase — how long it takes the signal to return to 50% of peak after the maximum. Shorter means faster clearance.
Rise Rate of rise ΔF/F₀ / s The slope of the rising phase of the calcium transient. A steeper rate means the signal climbs faster.

3.2 TG Metrics (Thapsigargin Protocol)

These metrics are specific to experimental protocols where thapsigargin (TG) is applied to deplete ER calcium stores. They only appear if your files contain TG response data.

TabFull nameUnitWhat it measures
TG PeakTG peak amplitudeΔF/F₀Maximum calcium release triggered by ER store depletion
TG SlopeTG initial slopeΔF/F₀ / sSteepness of the initial TG-induced calcium rise
TG AUCTG area under curveΔF/F₀ · sTotal calcium flux during the TG response

3.3 Add-Back Metrics

These metrics measure the calcium entry that occurs when extracellular calcium is re-added after TG-induced ER depletion (store-operated calcium entry, SOCE).

TabFull nameUnitWhat it measures
AB PeakAdd-back peakΔF/F₀Peak calcium entry amplitude during add-back phase
AB SlopeAdd-back initial slopeΔF/F₀ / sRate of calcium entry at the start of add-back
AB AUCAdd-back area under curveΔF/F₀ · sTotal calcium entry during add-back response
AB TTPAdd-back time to peaksTime from add-back until peak calcium entry is reached

3.4 Trace Tabs (Raw F and ΔF/F₀)

The first two tabs show time-series traces rather than scalar summary statistics:

  • Raw F: Shows absolute fluorescence intensity over time for every ROI in every file. Traces are grouped and color-coded by condition, with individual replicates overlaid.
  • ΔF/F₀: Shows the normalized fluorescence change relative to the resting baseline. This is the standard way to compare calcium signals across cells and experiments independent of baseline fluorescence levels.

For trace tabs, the X axis range control (in the Appearance panel) allows you to zoom into a specific time window.

ℹ Trace availability
Trace tabs are only shown if the uploaded files contain Raw_Traces or DeltaF sheets. Files exceeding the 500,000-cell trace limit are noted as "traces skipped" in the file list but their Metrics data is still available.
Section 4

Chart Styles

Five visualization types for metric tabs — choose the one that best tells your data's story

The chart style selector is located directly below the tab bar. Changing style updates all metric plot tabs instantly. Trace tabs always show overlaid time-series regardless of the selected style.

Chart Style Selector
Violin
Bar
Box
Strip
Raincloud
StyleWhat it showsBest used when
Violin A mirrored kernel density estimate showing the full probability distribution of values within each condition. Individual ROI points are overlaid as a scatter plot. Per-file means are shown as larger dots. The overall condition mean ± error bar is drawn on top of the violin. You want to show the full data distribution and identify skewness, bimodality, or outliers. This is the default and recommended style for most calcium imaging data.
Bar A classic bar chart showing the condition mean as bar height, with error bars (SEM, SD, or 95% CI) drawn on top. Individual ROI points are overlaid. You need a compact, traditional visualization for a narrow figure panel, or when reviewers expect bar charts.
Box A box-and-whisker plot showing the median (center line), interquartile range (box), Tukey whiskers (1.5× IQR), and outlier points beyond the whiskers. You want to highlight the median and spread without assuming a normal distribution, or when comparing skewed datasets.
Strip All individual data points shown as a vertical scatter plot, jittered horizontally to reduce overplotting. Per-file means are shown as larger dots. No distributional shape is estimated. You have small sample sizes where individual points are meaningful, or you want the most transparent data display.
Raincloud A "raincloud" combines a half-violin (density estimate) above with a strip plot of individual points below. Per-file means are overlaid. This style maximizes information density. You want both the distributional shape and every individual point visible — ideal for publication figures.
💡 Which style to choose?
For publication figures, Raincloud or Violin are recommended as they show the full data distribution alongside the summary statistics, satisfying transparency requirements from most journals. Bar charts alone are considered insufficient by many reviewers.
Section 5

Plot Controls

Real-time controls in the right panel that adjust how data is summarized and displayed

All controls in the Plot Controls panel take effect immediately — plots refresh within milliseconds of any change. Your settings are automatically saved and restored on your next visit.

5.1 Summary Statistics

The Summary stat dropdown determines what the central tendency measure represents:

  • Mean: Arithmetic mean of all values within the condition. Sensitive to outliers. Recommended for normally distributed data.
  • Median: The middle value when data is sorted. Robust to outliers and skewed distributions. Recommended when data is non-normal.

5.2 Error Bars

The Error bars dropdown controls the vertical extent of error bars on the summary statistic:

OptionFull nameHow it is calculatedRecommended when
SEMStandard Error of the MeanSD / √n (where n = number of replicates)You want to show uncertainty of the mean estimate across replicate experiments
SDStandard DeviationSquare root of variance across replicate meansYou want to show the spread of replicate measurements
95% CI95% Confidence IntervalMean ± 1.96 × SEMYou want inferential interval bars that scale with sample size
NoneNo error bars—You prefer minimal clutter or are relying on the distribution shape alone
ℹ Error bars are computed across replicate means
Error statistics (SEM, SD, CI) are always computed across the per-file means, not across individual ROIs. This correctly treats each experiment as one biological replicate — the recommended approach for calcium imaging superplots.

5.3 Data Representation Checkboxes

ControlWhat it does
Paired lines Draws connecting lines between per-file means across conditions. Use this when your experiments are paired — e.g., the same cell line measured under two different conditions on the same day.
Replicate dist. Instead of showing all individual ROI values, shows only the distribution of per-file means. Useful when you have many ROIs per file and want to emphasize the replicate-level variability rather than ROI-level variability.
Log axis Switches the Y axis to a logarithmic scale. Useful when comparing data that spans multiple orders of magnitude, such as event frequency.
Horizontal Rotates the chart 90° so conditions are listed on the Y axis and values run left-to-right. Useful when condition names are long, or when you prefer a horizontal layout.
Grid Shows or hides horizontal gridlines behind the plot. Uncheck for cleaner figures.
Conditions legend Shows or hides the condition color legend in the Appearance panel.
Replicate legend Shows or hides the replicate symbol/color legend in the Appearance panel.

5.4 Condition Ordering

The Order dropdown controls the left-to-right sequence of conditions on the X axis:

  • Entered: Conditions appear in the order they were first assigned (default).
  • Manual: You control the order yourself using ↑/↓ arrow buttons that appear next to each condition in the legend panel. Drag the order to match your preferred layout.
  • A–Z: Alphabetical order by condition name.
  • By summary: Sorted by ascending mean or median value — useful for ranking conditions.

5.5 Point Style Sliders

SliderRangeDefaultEffect

Section 6

Appearance Panel

Customizing colors, labels, fonts, and axis ranges for publication-ready figures

The Appearance panel is located below the Plot Controls in the right sidebar. All changes apply instantly to the active plot and are saved in the browser automatically.

6.1 Condition Colors

Color Palette

The Palette dropdown offers two preset color sequences:

Default palette (12 colors)

A vibrant 12-color sequence optimized for general use on white or dark backgrounds.

Okabe-Ito palette (8 colors)

A colorblind-safe palette (safe for protanopia, deuteranopia, and tritanopia). Recommended for figures intended for wide publication.

Per-Condition Color Override

Each condition in the legend has a color swatch and a hex color input. Click the swatch to open a color picker, or type a hex code directly. The ↺ reset button next to each condition restores its palette default.

Manual Condition Ordering

When Order is set to "Manual" (in the Controls panel), ↑ / ↓ arrow buttons appear next to each condition in the legend. Click them to move conditions up or down in the X-axis sequence. The order is saved automatically.

6.2 Labels & Title

Click Reset ranges to clear all custom axis limits and return to auto-scaling.

⚠ Session data is not persisted

Section 10

Dark / Light Theme

Switching between light and dark mode

A theme toggle button is located in the top-right of the header bar. It reads "Dark mode" when you are currently in light mode (click to switch to dark), and "Light mode" when in dark mode (click to switch to light).

Dark mode applies across the entire interface — panels, plots, legends, and all UI controls adopt a dark color scheme automatically. Plot backgrounds, gridlines, and axis labels update to maintain readability.

Your theme preference is stored in the browser and restored on every visit.

💡 Note on exported figures
The plot background color in exported PNG files matches the current theme. If you need white-background figures for publication, make sure to export in light mode.

Section 12

Troubleshooting

Common issues and how to resolve them
ProblemLikely causeSolution
No plots appear after assigning conditions The condition name has only been typed for one file, or the Metrics sheet is missing from the uploaded files. Make sure at least one condition has more than zero files with that condition name assigned, and verify the .xlsx file contains a "Metrics" sheet.
Upload fails with "bad_structure" The file is not a valid Excel workbook, or it was exported from unsupported software. Open the file in Excel, verify it has the expected sheet structure (Metrics, Raw_Traces, or DeltaF), and re-export from the Ca²⁺tch-One.
Upload fails with "per-file RAM cap" The file has too many ROIs or too many time points for the current session memory budget. Remove some existing files to free up session RAM, or reduce the recording to fewer ROIs/frames before exporting.
Trace tabs do not appear Uploaded files lack Raw_Traces or DeltaF sheets, or the trace sheets exceeded the 500,000-cell limit and were skipped. Check the file list for "traces skipped" warnings. If skipped, reduce the number of time points or ROIs in the export.
Condition auto-restore shows wrong names The file was renamed since the last session, or the browser localStorage was cleared. Re-assign condition names manually. New assignments will be saved for the next session.
Some metric tabs are missing The loaded files do not contain those particular metrics in their Metrics sheet. Tabs are hidden automatically for unavailable metrics. This is normal — a "Hidden metrics" note will appear in the controls area if metrics differ across files.
Server does not start Python or required packages are not installed, or the port 5000 is in use by another application. Check that Python 3.9+ is installed and run pip install -r requirements.txt in the backend folder. If the port is occupied, close the other application or edit the start script to use a different port.
Plot looks different after refreshing Files are not loaded (session data is in-memory only) but preferences are restored from localStorage. Re-upload your files. Condition names will be restored automatically from the previous session.
💡 Starting fresh
To reset all preferences and start with a clean slate: open your browser's Developer Tools, go to Application → Storage → Local Storage, and clear the entries for localhost:5000. This removes all saved settings including colors, typography, and condition assignments.
